截屏功能修复说明
==================

问题描述：
----------

用户反馈截屏和小白程序被意外结束。

根本原因：
----------

1. **截图窗口关闭逻辑问题**：在保存对话框中取消保存时，截图窗口会被错误关闭
2. **选区太小处理不当**：当用户选择的区域太小时，截图窗口直接关闭而不是让用户重新选择
3. **宠物窗口显示问题**：截图取消后，宠物窗口可能无法正确恢复显示
4. **缺少信号机制**：无法准确捕获截图操作的完成状态

修复内容：
----------

### 1. 添加截图完成信号
```python
# 定义截图完成信号
capture_finished = pyqtSignal()
```
在截图操作完成（无论是保存成功、取消还是出错）时发送信号。

### 2. 优化选区太小处理
```python
# 当选区太小时，不是关闭窗口，而是重置选择状态
QMessageBox.warning(self, "提示", "选择区域太小，请重新选择")
self.is_selecting = False
self.start_point = QPoint()
self.end_point = QPoint()
self.update()
```
用户可以选择新的区域，而不是被迫关闭截图窗口。

### 3. 优化保存对话框取消逻辑
```python
if cropped_image.save(file_path):
    pass
else:
    QMessageBox.warning(self, "截图失败", "保存图片失败")
    return  # 只有保存失败时才return，不会关闭窗口
```
如果用户取消保存对话框，截图窗口会保持打开，用户可以继续操作。

### 4. 添加showPet方法
```python
def showPet(self):
    """显示宠物窗口"""
    self.show()
    self.activateWindow()
    self.raise_()
```
确保宠物窗口能够正确显示并激活。

### 5. 优化截图完成后的信号发送
- **保存成功时**：发送信号显示宠物
- **保存失败时**：发送信号显示宠物
- **右键取消时**：发送信号显示宠物
- **ESC取消时**：发送信号显示宠物

使用方法：
----------

### 基本使用流程

1. 右键点击小白宠物
2. 选择"工具" → "截屏"
3. 宠物窗口会自动隐藏
4. 屏幕上会出现半透明遮罩
5. 使用鼠标左键拖动选择要截取的区域
6. 选择完成后会弹出保存对话框
7. 选择保存路径并点击"保存"
8. 截图自动保存，宠物窗口恢复显示

### 交互操作

| 操作 | 功能 |
|------|------|
| **鼠标左键拖动** | 选择截取区域 |
| **鼠标右键点击** | 取消截图，恢复宠物显示 |
| **ESC键** | 取消截图，恢复宠物显示 |
| **Enter键** | 确认当前选区并打开保存对话框 |
| **选择区域太小** | 提示用户重新选择（不关闭窗口） |

### 高级功能

1. **实时尺寸显示**：选择区域时会实时显示宽度和高度
2. **四角标记**：选择区域四角有明显的蓝色标记点
3. **智能位置调整**：尺寸标签会自动调整位置避免超出屏幕

使用建议：
----------

1. **截图前**：确保要截取的内容在屏幕上可见
2. **选择区域**：拖动时可以从任意角落或边缘开始
3. **保存位置**：默认保存到用户图片文件夹，可自定义路径
4. **格式选择**：支持PNG和JPG格式，建议使用PNG保留更多细节

常见问题：
----------

Q: 为什么选区太小时不直接保存？
A: 太小（小于5×5像素）的选区没有实际意义，因此提示用户重新选择。

Q: 可以截取多个区域吗？
A: 当前版本只支持单区域截图。如需多个区域，请多次使用截屏功能。

Q: 截图保存在哪里？
A: 默认保存在用户图片文件夹（Windows为Pictures文件夹）。

Q: 为什么截图时宠物会隐藏？
A: 这是为了避免截图包含宠物自身。如果需要截取包含宠物的画面，可以先截图再调整宠物位置。
